# PickUp - フットサルマッチングプラットフォーム 企画書

## プロダクト概要

都市部でフットサルの仲間を見つけるマッチングプラットフォーム。
個サル文化をベースに、レベル別・エリア別で簡単にマッチに参加・主催できる。

## なぜフットサルから始めるか

- **流動性が命** → 1競技に集中してユーザー密度を確保
- **5vs5で少人数成立** → マッチングの最小単位が小さい
- **屋内施設多数** → 天候リスクが低い
- **用具ハードル低** → シューズだけでOK（貸出もあり）
- **都市部需要集中** → 渋谷・新宿エリアで密度が作れる
- **個サル文化が既存** → 「知らない人と蹴る」行動が定着済み

## MVP機能設計

### ユーザー側
- エリア検索
- 日時フィルター
- レベル選択（初心者OK / 経験者 / ガチ）
- ポジション希望
- 参加ボタン（即決済 ※Phase 2以降）
- レビュー機能
- キャンセル規定（24h前無料、以降50%）

### 主催者側
- 募集投稿作成
- 必要人数設定
- 料金設定
- 承認 / 自動承認
- メンバー評価
- 当日QRチェックイン

### 安全設計
- 顔写真登録必須
- 本人確認（SMS）
- レビュー履歴可視化
- ドタキャン罰金制度

## マネタイズ戦略

| 収益源 | 詳細 | タイミング |
|--------|------|-----------|
| 参加手数料 15% | 例: ¥2,000 → ¥300手数料、月150試合で225万円規模 | Phase 2 |
| 月額会員 ¥980 | 手数料半額、優先マッチング | Phase 2 |
| B2B施設提携 | 月額3-5万/施設、中期安定収益源 | Phase 3 |

### 課金導入のトリガー（3条件）
1. 週10マッチ以上が安定成立
2. リピート率30%以上
3. ユーザーから「アプリ内決済」の声が出る

**Phase 0-1は無料運用**（参加費は現地手渡し or PayPay）。
流動性の確保が最優先。早すぎる課金は参加ハードルを上げてマッチ不成立リスク。

## アイデアメモ

### 条件付き予約機能
- 個人やサークルが施設予約をする際、**人数が集まった時点で予約確定**になる機能
- 人数未達の場合 → **48時間前に自動キャンセル**（施設側が再販可能）
- 施設側メリット: 「確実に来る団体」のみ予約 → 空予約リスク減
- → B2B施設提携（Phase 3）への自然な導線になる
